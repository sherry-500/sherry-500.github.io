Simulation methods: Particle filtering

sherry-500
2025-08-01
Statistics
Machine Learning, Statistics

在需要从一个维度随时间增长的后验分布π(θ0 : t ∣ y0 : t)中采样的场景中，即使后验分布已知，实现一个采样策略的时间复杂度也会随着t线性增长。这使得以批模式处理的MCMC方法在新的信息可用时需要完全重新运行。因此，本文将介绍一种顺序算法，它会随着新数据的不断获取而逐步运行。这些算法更快也更适用于需要实时更新的场景（在线模式）。

Dynamic system

首先，我们先考虑动态系统在状态空间中的表示。系统中有一个不可观测的状态向量θt ∈ ℝK，和一个可观测的变量Yt, t = 0, 1, …，具体的：

  1. θt是一个马尔可夫过程，即 定义θt的所有相关信息都包含在θt − 1中。

  2. zai对于所有的s < t，Yt ⟂ Ys ∣ θt，即在给定实际状态向量的情况下，可观测变量和它们的历史之间是独立的。

Sequential importance sampling

我们先从sequential importance sampling（SIS）算法开始，它是对IS算法的一种修改，用于在不改变历史轨迹{θ1 : t − 1(s), s = 1, 2, …, S}的前提下计算π(θ0 : t ∣ y0 : t)的估计，核心思想是使用如下形式的提议分布 该提议分布允许顺序地计算权重 令，被称为incremental importance weight。 这种算法具有保持固定计算复杂度的优良特性。

我们顺序地获取抽样θt(s)，θt(s)被称为粒子（particle）。具体的，θ0(s)是在t = 0时刻从q(θ0)中抽取的样本，接下来，在时刻t = h，从q(θh ∣ θh − 1, yh)中抽取样本θh(s)。

SIS在短期内对从后验分布中抽样是有效的，但是因为它是IS的一种特例，SIS也继承了IS的缺点。实际上，随着时间t的增加，重要性权重的方差会指数级增长。这意味着重要性权重在长期内趋于退化，也就是说，所有的概率质量集中在少数几个权重上，这种现象被称为样本贫化（sample impoverishment）或权重退化（weight degeneracy）。这种现象在动态线性模型的例子中也可以观察到，例如，在t = 50时，最高的标准化权重为53%，而5000个粒子中有7个占据了总概率的87%。

Particle filtering

鉴于此，在实际中，我们通常采用sampling/importance resampling（SIR）的方法。这种策略能避免由于重置系统而导致的误差累积。Bootstrap filter算法在每个时刻重采样，重采样是通过使用标准化权重作为选择概率，从当前粒子集合中抽取S个粒子来进行的。这保证了权重小的粒子被选中的概率低。经过重采样，标准化权重设置为1/S。Bootstrap filter设计SIR算法的多次的迭代，这意味着重采样之后的轨迹不再是独立的。这种多项式重采样提供了对通过SIS获得的后验分布的无偏近似。

在上面的代码实现中我们在每个时刻都进行重采样，但是，在粒子的有效样本容量下降到一个特定的阈值以下时才进行重采样是一种更常见的做法。并且，我们应该使用{θ1 : t(s), wt*(s)}估计后验分布，因为相比于基于{θ1 : tr(s), 1/S}的计算，这种做法有更低的Monte Carlo error。

  • 快速并且易于实现
  • 具备模块化特性，简单地调整重要性分布和权重的表达式就能适应问题的更改
  • 可以并行计算
  • 实现里复杂模型的直接顺序推断

同时，也有一些不足：

  • 重采样的步骤引入了额外的Monte Carlo变异
  • 使用状态转移分布（先验分布）作为重要性分布，经常会导致不好的表现，表现在对观测序列缺乏鲁棒性
